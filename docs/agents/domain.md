# Domain docs

This repo uses a **single-context** layout.

## Files

- **`CONTEXT.md`** (repo root) — domain vocabulary, concepts, and conventions. Read this first when onboarding to the project.
- **`docs/adr/`** — Architecture Decision Records. Each ADR is a numbered markdown file (e.g. `0001-use-xyz.md`).

## Consumer rules

1. Read `CONTEXT.md` before making architectural or domain-sensitive changes.
2. Check `docs/adr/` for relevant past decisions before proposing new ones.
3. If `CONTEXT.md` does not yet exist, consider creating it as part of the next domain-learning task.
4. If `docs/adr/` does not yet exist, create it when the first architectural decision is recorded.
